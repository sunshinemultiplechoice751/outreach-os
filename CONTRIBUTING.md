# Contributing to outreach-os

Thank you for your interest in contributing! This project is open to all contributions.

## Ways to Contribute

- **Report bugs** — Open an issue describing the problem
- **Suggest features** — Open an issue with your idea
- **Submit workflow improvements** — Fork, edit, and open a PR
- **Improve documentation** — Fix typos, add examples, improve setup guides
- **Add new workflow integrations** — WhatsApp, Slack, HubSpot, etc.

## Development Setup

1. Fork this repo
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/outreach-os.git`
3. Create a branch: `git checkout -b feat/your-feature-name`
4. Make your changes
5. Commit with a clear message: `git commit -m "feat: add WhatsApp follow-up workflow"`
6. Push: `git push origin feat/your-feature-name`
7. Open a Pull Request

## Commit Convention

We use conventional commits:
- `feat:` — new feature or workflow
- `fix:` — bug fix
- `docs:` — documentation changes
- `refactor:` — code restructuring
- `chore:` — maintenance

## Pull Request Guidelines

- Keep PRs focused on a single change
- Include a clear description of what changed and why
- For new workflows: include the JSON file, update README's workflow table, and add setup notes
- Test your workflow in N8N before submitting
